# Demo V9 變更摘要

## 首頁

- 未簽收公告新增預覽區。
- 首頁只顯示前 2 筆公告與前 2 筆任務，其餘折疊展開。
- 主管待辦總覽只顯示未完成數最多的前 2 組，其餘折疊。
- 「今日假患者」改為「今日患者」。

## 今日流程看板

- 每個流程欄位只顯示前 2 位患者，其餘折疊展開。
- 患者卡片可點開查看檔案。
- 「下一步」改為「完成本站 → 下一步」，用來模擬各站人員完成處理後推進流程。

## 患者管理

- 患者主檔改名為患者管理。
- 補入新增個案、初診建檔、複診登記、個案查詢、列管、客服關懷等入口。
- 患者檔案以分層欄位呈現：基本資料、就診流程、居家與製衣。

## 公告與簽收

- 新增公告表單：範圍、分類、標題、內容、是否需簽收。
- 支援總公司公告、院區公告與組別公告。

## 數位足跡稽核

- 改成帳號、模組、動作、關鍵字篩選。
- 大量資料放入折疊區，避免頁面過重。

## 人員與權限

- 新增虛擬員工管理，用於訓練、排班與分組測試。
- 可建立各組 2 至 3 位虛擬員工。
- 可調整虛擬員工姓名、職稱、角色、組別與預設院區。

## 需要執行 SQL

請到 Supabase SQL Editor 執行：

```text
supabase/v9_patch.sql
```
