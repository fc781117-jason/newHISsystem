# NEW HIS Demo V14 整合版說明

## 本版定位

V14 是把前面 V12、V13、V14 三個階段直接合併的大版，方便一次交給不同組別測試。

## 三大測試組別

### 一、現場流程組

測試頁面：

- 工作指揮中心
- 今日流程看板
- 掛號與初複診
- 患者管理

測試重點：

- 新增個案
- 初診建檔
- 複診登記
- 未預約報到
- 取消原因
- 等待看診
- 等待治療
- 等待檢查
- 等待開處方
- 等待拿藥
- 完診流程

### 二、個案照護組

測試頁面：

- 居家／製衣／客服
- 任務中心
- 患者檔案
- OpenAI 行政助理

測試重點：

- 居家諮詢
- 居家聯繫
- 居家動態
- 保健衣劃記
- 製衣工單
- 等待寄送
- 客服關懷
- 列管追蹤
- 工單與任務連動

### 三、行政營運組

測試頁面：

- 營運報表
- 人員與權限
- 請假與代理
- 月排班與調班
- 公告與簽收
- 數位足跡稽核
- 功能整併總覽
- Demo 管理設定

測試重點：

- 角色權限
- 虛擬員工
- 月排班
- 調班申請
- 請假卡位
- 公告簽收
- 稽核篩選
- 營運報表

## 必做 SQL

請到 Supabase SQL Editor 執行：

supabase/v14_patch.sql

這會建立：

- demo_staff
- patient_events
- appointments_mock
- work_orders
- shift_swap_requests

並刷新 schema cache。

## 注意

本版仍為 Demo 與訓練環境，不可輸入真實病患資料。
