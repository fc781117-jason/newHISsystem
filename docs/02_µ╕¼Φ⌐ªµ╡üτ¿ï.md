# Demo V7 測試流程

## 1. Google 登入測試

- 使用 `fc781117@gmail.com` 登入。
- 確認角色為最高系統管理員。
- 使用另一個 Google 帳號登入。
- 確認會自動建立人員檔。

## 2. 雲端資料測試

- 最高管理員登入後，點「重置 50 筆」。
- 用第二個帳號登入。
- 確認第二個帳號看得到同一批資料。

## 3. OpenAI 行政助理測試

- 先部署 Supabase Edge Function。
- 設定 `OPENAI_API_KEY`。
- 進入 OpenAI 行政助理。
- 點「呼叫 OpenAI 產生建議」。
- 確認產出行政交班摘要。
- 確認內容不涉及醫療診斷或治療決策。
- 若尚未部署，可點「本機摘要備援」。

## 4. 月排班測試

- 進入月排班與工時。
- 選擇本月、下月或未來月份。
- 點「產生整月排班」。
- 檢查月排班表是否產生整月資料。
- 檢查月工時統計與公平性提示。

## 5. 三個月內請假卡位測試

- 進入請假與代理。
- 點本月、下月、二個月後、三個月後卡位。
- 檢查假單月份、代理人、代理狀態、審核狀態與進度。

## 6. 今日流程測試

- 進入今日流程看板。
- 點患者卡片「下一步」。
- 用另一個帳號重新整理。
- 確認狀態已更新。

## 7. 任務測試

- 新增雲端假任務。
- 完成任務。
- 到稽核頁查詢操作紀錄。

## 8. 權限測試

- 最高管理員進入「人員與權限」。
- 將某帳號改成治療師、客服或稽核者。
- 重新登入該帳號。
- 確認可用模組不同。

## 9. 公告簽收測試

- 新增公告。
- 用另一個帳號登入簽收。
- 到稽核頁查詢公告簽收紀錄。

## 10. 動態浮水印測試

- 開啟任意頁面。
- 觀察畫面背景是否有帳號、角色、院區與時間。
