# 平台申請與部署教學：Supabase + Google Login + OpenAI + GitHub + Vercel

## 本版需要準備的資料

```text
1. Supabase Project URL
2. Supabase anon public key
3. Google OAuth Client ID
4. Google OAuth Client Secret
5. OpenAI API Key
6. GitHub repository
7. Vercel project
```

目前仍不需要：

```text
Line API
Email API
SMS API
正式地端伺服器
真實患者資料
```

---

## A. 建立 Supabase 專案

1. 登入 Supabase。
2. 點 `New project / 新專案`。
3. Project name / 專案名稱：例如 `nrs-demo-v7`。
4. Database Password / 資料庫密碼：自行設定並保存。
5. Region / 區域：選擇離台灣較近的區域。
6. 點 `Create new project / 建立新專案`。

---

## B. 執行資料庫 SQL

1. 進入 Supabase Project。
2. 左側點 `SQL Editor / SQL 編輯器`。
3. 點 `New query / 新查詢`。
4. 打開本專案的 `supabase/schema.sql`。
5. 全部複製貼上。
6. 點 `Run / 執行`。

---

## C. 取得 Supabase 環境變數

1. 左側點 `Project Settings / 專案設定`。
2. 點 `API`。
3. 複製：
   - `Project URL`
   - `anon public key`

建立 `.env.local`：

```env
VITE_SUPABASE_URL=你的 Project URL
VITE_SUPABASE_ANON_KEY=你的 anon public key
VITE_APP_ADMIN_EMAIL=fc781117@gmail.com
```

---

## D. 設定 Google OAuth

### D1. Google Cloud Console

1. 開啟 Google Cloud Console。
2. 建立專案：`New Project / 新增專案`。
3. 進入 `APIs & Services / API 與服務`。
4. 點 `OAuth consent screen / OAuth 同意畫面`。
5. 選擇 External / 外部 或 Internal / 內部。
6. 填寫 App name / 應用程式名稱，例如 `NRS Demo V7`。
7. 儲存。

### D2. 建立 OAuth Client

1. 進入 `Credentials / 憑證`。
2. 點 `Create credentials / 建立憑證`。
3. 選 `OAuth client ID / OAuth 用戶端 ID`。
4. Application type / 應用程式類型：選 `Web application / 網頁應用程式`。
5. Authorized JavaScript origins / 已授權的 JavaScript 來源：
   - 本機：`http://localhost:5173`
   - Vercel 網址：部署完成後補上
6. Authorized redirect URIs / 已授權的重新導向 URI：
   - 到 Supabase Auth Providers Google 頁面複製 callback URL。
   - 通常格式類似：`https://你的-project-ref.supabase.co/auth/v1/callback`
7. 建立後複製：
   - Client ID
   - Client Secret

---

## E. 在 Supabase 啟用 Google Provider

1. 回到 Supabase。
2. 左側點 `Authentication / 驗證`。
3. 點 `Providers / 提供者`。
4. 找到 `Google`。
5. 開啟 `Enable Sign in with Google / 啟用 Google 登入`。
6. 貼上 Google Client ID。
7. 貼上 Google Client Secret。
8. 儲存。

---

## F. 設定 OpenAI API Key

### F1. 到 OpenAI Platform 建立 API Key

1. 登入 OpenAI Platform。
2. 進入 API Keys。
3. 建立新的 Project API Key。
4. 複製 API Key。
5. 不要貼在前端 `.env.local`，也不要上傳 GitHub。

### F2. 安裝 Supabase CLI

本機需要 Supabase CLI 才能部署 Edge Function。

### F3. 登入 Supabase CLI

```bash
supabase login
```

### F4. 連結你的 Supabase Project

```bash
supabase link --project-ref 你的-project-ref
```

Project ref 可以在 Supabase 專案網址或 Project Settings 裡找到。

### F5. 設定 Supabase Secret

```bash
supabase secrets set OPENAI_API_KEY=你的_OPENAI_API_KEY
supabase secrets set OPENAI_MODEL=gpt-5.4-mini
```

### F6. 部署 Edge Function

```bash
supabase functions deploy ai-admin-assistant
```

## G. 設定 Supabase Redirect URLs

1. Supabase 左側點 `Authentication / 驗證`。
2. 點 `URL Configuration / URL 設定`。
3. Site URL / 網站網址：
   - 本機測試：`http://localhost:5173`
   - 部署後：Vercel 網址
4. Redirect URLs / 重新導向網址：
   - `http://localhost:5173`
   - `https://你的-vercel-domain.vercel.app`

## I. 上傳 GitHub

1. 到 GitHub。
2. 點 `New repository / 新增儲存庫`。
3. 建議設為 `Private / 私有`。
4. 上傳整個專案資料夾。
5. 不要上傳 `.env.local`。
6. 不要上傳任何 OpenAI API Key。

---

## J. 部署 Vercel

1. 登入 Vercel。
2. 點 `Add New... / 新增`。
3. 選 `Project / 專案`。
4. 匯入 GitHub repository。
5. Build Command / 建置指令：`npm run build`
6. Output Directory / 輸出資料夾：`dist`
7. 到 `Environment Variables / 環境變數` 加入：
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   - `VITE_APP_ADMIN_EMAIL`
8. 點 `Deploy / 部署`。

OpenAI API Key 不放 Vercel，放 Supabase Function Secret。

---

## K. 部署後補設定

部署完成後會得到 Vercel 網址。

請回到：

1. Google Cloud Console：
   - Authorized JavaScript origins 加入 Vercel 網址。
2. Supabase：
   - Site URL 改成 Vercel 網址。
   - Redirect URLs 加入 Vercel 網址。
