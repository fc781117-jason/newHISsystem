# Demo V6 邊界與資安提醒

## 本版已接近真實互動

本版資料存在 Supabase 雲端資料庫。多人可以登入同一個 Demo 系統，共同操作假資料。

## 本版新增 OpenAI API

OpenAI API Key 不得放在前端，也不得上傳 GitHub。  
本版採 Supabase Edge Function 方式呼叫 OpenAI，API Key 存放在 Supabase Secrets。

## 仍然不是正式醫療系統

不可放入：

- 真實患者姓名
- 真實電話
- 真實病歷
- 真實生日
- 真實地址
- 真實合約
- 真實醫囑
- 真實藥品資料

## 正式版仍需加強

- 更嚴格的 RLS
- 院區資料隔離
- 敏感個案權限
- OpenAI 輸入資料去識別化
- AI 使用紀錄封存
- 操作紀錄不可刪除
- 備份與還原
- 日誌封存
- 資安測試
- 法規檢核
