# Demo V7 變更摘要

## 本版修正

1. 手機版頁面重排，避免上方列與內容重疊。
2. 首頁移除「追加 20 筆／重置 50 筆」，統一移到「Demo 管理設定」。
3. 首頁新增登入者身份卡，呈現姓名、職稱角色、組別、預設院區。
4. 新增「功能整併總覽」，依原系統功能重新歸類。
5. 月排班新增「調班申請」區塊。
6. 新增 `shift_swap_requests` 資料表與 V7 patch SQL。

## 原系統功能整併方向

### 今日現場流程

原功能包含：等待看板、等待看診、等待治療、等待檢查、等待開處方、等待拿藥、初診、複診。

新版承接：今日流程看板、患者主檔、任務中心。

### 居家、保健衣、製衣物流

原功能包含：等待居家諮詢、居家聯繫、居家動態表、居家總覽、居家未完成、居家製衣、保健衣未完成、保健衣製衣、等待寄送、製衣前確認。

新版承接：任務中心、製衣物流、居家服務狀態、跨組別待辦。

### 客服、醫事、藥局、統計與公告

原功能包含：客服管理、工作聯繫、公告、列管事項、醫事管理、藥局管理、統計管理、專案系統、人事系統。

新版承接：公告簽收、數位足跡、月排班、請假代理、人員權限、統計報表。

## 已部署 V6 者的更新方式

1. 先到 Supabase SQL Editor 執行：`supabase/v7_patch.sql`。
2. 將 GitHub 專案中的檔案覆蓋為 V7。
3. Commit 後 Vercel 會自動重新部署。
4. 手機重新開啟網站測試版面。
